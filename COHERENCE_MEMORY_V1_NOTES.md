# Coherence Memory Engine v1

Adds long-range personalization on top of coherence state, interference state, behavior adaptation, and trajectory.

## What it does

- Tracks recurring drift signatures over the previous 21 days.
- Detects repeated resistance to assigned practices.
- Detects repeated completion/mastery patterns.
- Estimates recovery velocity per movable frequency body.
- Determines expansion tolerance.
- Feeds memory bias into the Today Engine.

## Philosophy alignment

Source remains the fixed core reference. Memory is calculated only against the movable bodies:

- Form
- Field
- Mind
- Code

The memory layer does not treat Source as damaged, weak, or drifting. It remembers how the movable bodies repeatedly destabilize or stabilize relative to Source.

## After deploy

Clear the frozen daily plan:

```js
localStorage.removeItem('q_today_plan')
location.reload()
```

The first few days may show baseline-building language. Strong memory behavior emerges after several daily plan snapshots exist.
