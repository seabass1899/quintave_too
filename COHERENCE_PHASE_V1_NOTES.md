# Coherence Phase Engine v1

Adds a macro-state layer that classifies the user into one of five developmental operating phases:

- Collapse / Rebuild
- Recovery
- Stabilization
- Expansion
- Integration

This phase does not replace Source anchoring, coherence-state, interference-state, trajectory, or memory. It sits above them and gives the Today Engine a macro-developmental bias.

## Key principles

- Source remains fixed and reference-only.
- Form, Field, Mind, and Code are the movable bodies.
- Recovery/collapse phases lower friction.
- Stabilization favors repeatable coherence.
- Expansion allows slightly deeper practice only after enough stability.
- Integration favors maintenance and continuity.
